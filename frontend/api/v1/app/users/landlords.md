# Landlords API

Domain: `Users`

Base prefix:

`/api/v1/app/{company}/users/landlords`

## Endpoints

Currently implemented:

- `GET /landlords`
- `POST /landlords`
- `GET /landlords/{landlord}`
- `DELETE /landlords/{landlord}`

Route exists but is not implemented in controller:

- `PUT/PATCH /landlords/{landlord}`

## List Landlords

`GET /api/v1/app/{company}/users/landlords`

Supported query params:

- Filters:
  - `filter[id]`, `filter[name]`, `filter[email]`, `filter[phone]`, `filter[created_at]`
- Sort:
  - `sort=id,name,created_at` (prefix with `-` for descending)
- Include:
  - Not supported
- Select fields:
  - Not supported
- Pagination:
  - `per_page`, `page`

Example:

- `GET /api/v1/app/12/users/landlords?filter[name]=Muriithi&sort=-created_at&per_page=20`

Response notes:

- Returns paginated Laravel API resource collection:
  - `data` (array of landlords)
  - `links`, `meta` pagination objects

## Create/Elevate Landlord

`POST /api/v1/app/{company}/users/landlords`

Behavior:

- If `email` matches an existing user, backend elevates that user to landlord.
- If no user exists, backend creates a new user then elevates.
- For new users, backend auto-sets:
  - `portal = 3`
  - `terms = true`
  - `policy = true`

Request fields:

| Field                    | Required    | Type           | Allowed Values / Notes                                |
| ------------------------ | ----------- | -------------- | ----------------------------------------------------- |
| `email`                  | Yes         | string (email) | Used to check existing user                           |
| `name`                   | Conditional | string         | Required when creating a brand-new user               |
| `phone`                  | Conditional | string         | Required when creating a brand-new user               |
| `has_vat`                | No          | boolean        | Optional                                              |
| `tax_pin`                | Conditional | string         | Required if `has_vat=true`; unique in `users.tax_pin` |
| `withholds`              | No          | array          | Optional - Get from list of withholding taxes                                              |
| `contracts`              | No          | array          | Optional - Add management contracts as shown in the payload below - Will automatically create properties in there.                                             |
| `accounts`              | No          | array          | Optional - Add bank accounts as shown in the payload below.                                             |


Notes:

- Password is generated by backend for new users and never provided by frontend.
- Frontend should surface backend `422` validation messages directly.

## Delete Landlord

`DELETE /api/v1/app/{company}/users/landlords/{landlord}`

Behavior:

- Returns `404` if the target user is not in landlord group.
- Removes landlord group membership from user.
- If user has no other groups left, backend deletes the user account.

Example response:

```json
{
  "message": "Landlord deleted successfully",
  "user": {
    "id": 304,
    "name": "Joyce Wanjiku",
    "email": "joyce.wanjiku@example.com",
    "phone": "+254712345678"
  }
}
```

## Update Landlord

`PUT/PATCH /api/v1/app/{company}/users/landlords/{landlord}`

Route exists via `Route::apiResource(...)`, but `update` is not implemented in `LandlordController`.

## Status Enum (Returned in Resource)

Common values returned for `status`:

- `pending` (color: `secondary`)
- `active` (color: `success`)
- `suspended` (color: `warning`)
- `inactive` (color: `danger`)

## Errors

Use shared behavior in `docs/frontend/api/v1/app/README.md`:

- Show backend `4xx` messages and validation errors to users.
- Show generic fallback for `5xx` responses.
